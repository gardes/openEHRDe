Grundlagen

Architektur im Überblick

openEHR definiert eine klare, schichtenbasierte Architektur für elektronische Patientenakten. Jede Schicht hat eine klar definierte Aufgabe – von der technischen Speicherung bis zur semantischen Abfrage.

🔍

Schnittstelle

REST API

Die openEHR REST API bietet standardisierte Schnittstellen für alle wesentlichen Operationen – von der EHR-Verwaltung bis zu komplexen semantischen Abfragen.

  • Compositions speichern und abrufen
  • AQL-Abfragen ausführen
  • Demografische Daten verwalten
