Software-Entwicklung mit openEHR

Technische Ressourcen, Plattformen und Best Practices für die Integration von openEHR in Ihre Anwendungen.

Architektur im Überblick

openEHR definiert eine klare, schichtenbasierte Architektur für elektronische Patientenakten:

  • Informationsmodell – wie klinische Daten prinzipiell gespeichert werden
  • Archetyp-Formalismus – was dokumentiert wird (z. B. Blutdruckmessung)
  • AQL – Archetype Query Language als semantische Abfragesprache
  • REST API – standardisierte Schnittstellen für alle wesentlichen Operationen

Verfügbare Plattformen

Mehrere Open-Source- und kommerzielle Plattformen implementieren den openEHR-Standard:

EHRbase

Eine Java-basierte Open-Source-Implementierung, die vollständig den openEHR-Standard unterstützt. REST API, AQL-Support, Docker-ready.

github.com/ehrbase/ehrbase

Better Platform

Kommerzielle openEHR-Plattform mit umfangreichem SDK-Support.

Archetype Designer

Webbasiertes Tool zur Erstellung und Verwaltung von Archetypen und Templates.

tools.openehr.org/designer

REST API

Die openEHR REST API erlaubt:

  • EHR anlegen und verwalten
  • Compositions speichern und abrufen
  • AQL-Abfragen ausführen
  • Demografische Daten verwalten
